## <a id="toc-0001"></a>KNN 和 k-means 的区别

* **K-Means** 是**无监督学习**（只管把没标签的数据抱团分堆，找 $K$ 个中心）。
* **KNN** 是**有监督学习**（数据已经有了明确的标签，给你一个新来的未知数据，让你预测它属于哪一类）。

KNN 的底层逻辑极其简单，全宇宙通用一句中国古话就能概括：**“近朱者赤，近墨者黑。”**

## <a id="toc-0002"></a>KNN 简介

### <a id="toc-0003"></a>📝 机器学习进阶笔记：KNN (K-近邻算法)

#### <a id="toc-0004"></a>1. 核心思想：看你的朋友圈，就知道你是什么人
假设你们学校有两大阵营：“理科生（打代码、做高数）”和“文科生（写诗、看历史）”。
现在转学来了一个**新同学**，我们不知道他是文科还是理科。怎么办？
**KNN 的做法极其简单粗暴：** 看看全校离他“兴趣爱好（特征）”**最近的 $K$ 个同学**是哪个阵营的。

* 如果设定 $K=5$。
* 我们在全校找出了兴趣和新同学最相似的 5 个人。
* 发现这 5 个人里，有 4 个是理科生，1 个是文科生。
* **投票表决：4 > 1。** 结论：这个新同学大概率是**理科生**！

这就是 KNN 的全部过程，没有高深的模型，没有复杂的求导，主打一个“随大流”。

#### <a id="toc-0005"></a>2. 算法执行“三步走”
当给计算机一个未知的新数据点时，它是这样硬算的：
1.  **算距离**：算出这个“新点”到训练集中**所有已知点**的距离（通常用欧氏距离 $d = \sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2}$ ）。
2.  **挑邻居**：把算出来的距离从小到大排个序，挑出距离最近的 **$K$** 个点。
3.  **大投票**：
    * 如果是**分类问题**（比如预测文/理科）：这 $K$ 个邻居里，哪个类别多，新点就归哪一类（少数服从多数）。
    * 如果是**回归问题**（比如预测房价）：把这 $K$ 个邻居的房价加起来**求个平均值**，就是新房子的预测价格。

#### <a id="toc-0006"></a>3. 灵魂拷问：$K$ 值到底该定多大？
在 KNN 里，唯一需要你操心的参数就是这个 $K$。$K$ 值的选择，直接决定了模型的生死。

* **如果 $K$ 太小（比如 $K=1$）**：
    * **现象**：只看离他最近的那**唯一 1 个**人。
    * **后果**：**极容易过拟合（死记硬背）**。万一离他最近的那个人是个“奇葩”（比如明明是个理科生，偏偏极其热爱写诗的噪音数据），新同学就会被直接带偏。模型极其敏感，容错率极低。
* **如果 $K$ 太大（比如 $K=N$，也就是全校总人数）**：
    * **现象**：听取全校所有人的意见。
    * **后果**：**极度欠拟合（变成傻子）**。不用算了，全校理科生有 600 人，文科生 400 人。无论新来什么同学，全校投票的结果永远是理科生获胜。模型完全失去了预测能力。

**💡 经验法则：** $K$ 通常选一个**较小的奇数**（比如 3, 5, 7, 9），选奇数是为了防止投票时出现 2:2 平局的尴尬场面。实际工程中一般通过**交叉验证（Cross-Validation）**试出来哪个 $K$ 最好。

### <a id="toc-0007"></a>⚠️ 考研/面试高频刺客考点

关于 KNN，面试官最喜欢挖下面这三个坑，请务必背熟：

#### <a id="toc-0008"></a>刺客 1：“懒惰学习 (Lazy Learning)”的代表作
大部分机器学习算法（比如决策树、神经网络），拿到训练数据后，都要吭哧吭哧花好几天时间**“训练”**出一个模型参数，等测试数据来了，直接套公式秒出结果。
**KNN 是极其罕见的“懒汉”**。它**根本没有训练过程（训练时间为 0）**！它只是把所有数据死死存在硬盘里。等新数据一来，它才开始当场疯狂计算距离。
**代价：测试时极其缓慢，极其消耗内存。**

#### <a id="toc-0009"></a>刺客 2：必须要“特征归一化 (Normalization)”
如果你的数据有两个特征：【身高（1.5米~1.9米）】和【年收入（5万~50万）】。
如果你直接算欧氏距离，因为“年收入”的数值动辄相差几十万，它在距离公式里会**绝对碾压**身高的微小差异。身高这个特征就完全失效了。
**结论：用 KNN 之前，必须把所有特征强行缩放到同一个比例尺（比如 0 到 1 之间），做到众生平等。**

#### <a id="toc-0010"></a>刺客 3：维度灾难 (Curse of Dimensionality)
当特征维度极其高（比如有 10000 个特征）时，空间大到超乎想象，所有数据点之间的距离都会变得差不多远，“近邻”这个概念就失效了，KNN 的性能会发生雪崩。
