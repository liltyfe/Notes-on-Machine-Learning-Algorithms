## <a id="toc-001"></a>贪心算法

### <a id="toc-002"></a>一、 贪心算法的核心本质

贪心算法的字面意思已经暴露了它的性格：**极其短视，只顾眼前利益，绝不秋后算账（不回溯）。**

在每一步做决策时，它总是选择当前看起来最好、收益最大的那一个选项，期望通过一连串的“局部最优”操作，最终拼凑出“全局最优”的结果。

**它和 DP 的根本区别在哪？**
* **DP 是“深思熟虑”的全局统筹：** 在做决定时，它会考虑未来，会通过状态转移方程把所有可能性都推演一遍（哪怕是折叠过的穷举），确保万无一失。DP 允许“先苦后甜”。
* **贪心是“短视”的局部收割：** 拿到什么最好就拿什么，一旦做出了选择，就永远无法反悔。贪心绝不接受“先苦后甜”。

---

### <a id="toc-002"></a>二、 贪心算法的“生死线”：为什么有时会翻车？

贪心并不是万能的。很多时候，一堆局部最优加起来，并不等于全局最优。

为了让你瞬间领悟，我们直接搬出你刚学过的**背包问题**。

**场景 1：0-1 背包（只能用 DP，贪心会翻车）**
假设你的背包容量是 50kg。
* 物品 A：重 10kg，价值 60 （单价 6/kg）
* 物品 B：重 20kg，价值 100 （单价 5/kg）
* 物品 C：重 30kg，价值 120 （单价 4/kg）

如果用贪心策略（每次都挑单价/性价比最高的），你会怎么选？
你会先抢走 A（剩 40kg），然后再拿走 B（剩 20kg）。此时 C 已经装不下了。你的总收益是 **160**。

但真正的全局最优解是什么？是直接拿 B 和 C，刚好装满 50kg，总收益是 **220**。
贪心在这里翻车了，因为它过于迷恋 A 的高单价，导致剩下的空间被浪费了。

**场景 2：分数背包 (Fractional Knapsack)（完美适用贪心）**
规则变了：现在的物品不是铁疙瘩，而是金粉、大米，**可以被切碎装入**。

这时候贪心策略就成了绝对真理：
先装满单价最高的 A（10kg），再装满单价次高的 B（20kg）。此时背包剩 20kg，直接把 C 劈开，装入 2/3 的 C（获得价值 80）。
总收益 = 60 + 100 + 80 = **240**。这就是数学上的绝对最优解！

**结论：** 贪心算法能否成立，取决于问题是否具备**“贪心选择性质”**（即局部的最优选择，必须能保证导出全局的最优解，且不受未来选择的干扰）。

---

### <a id="toc-003"></a>三、 复试最爱考的贪心模型

贪心算法没有 DP 那么固定的套路（比如先定义状态再写方程），它的难点往往在于**想到那个贪心策略**，并证明它是对的。以下是复试中最常见的几个模型：

1.  **区间调度问题（Activity Selection）：** 给定 n 个会议的开始和结束时间，如何在一个会议室里安排尽可能多的会议？（**核心策略：永远挑结束时间最早的会议**）。
2.  **哈夫曼编码（Huffman Coding）：** 计网和数据结构中经常考，每次挑出现频率最低的两个节点合并来构建最优前缀树，这也是标准的贪心。
3.  **图论里的贪心：** 你之前复习 408 肯定背过的最小生成树（Prim 算法、Kruskal 算法）和单源最短路径（Dijkstra 算法），它们的底层思想其实全都是贪心——每次都找当前距离最近的点或权重最小的边。

## <a id="toc-004"></a>例题

### <a id="toc-005"></a>一、 经典考题：区间调度问题 (Interval Scheduling)

**问题描述：**
假设你是一个公司的行政，今天有 $N$ 个部门都要申请使用同一个会议室。每个会议都有一个开始时间 `start` 和结束时间 `end`。会议室在同一时间只能容纳一个会议。
**问：** 你最多能安排多少个会议，使得它们互不冲突？

---

### <a id="toc-006"></a>二、 核心破局点：怎么“贪”才是对的？

面对这个问题，直觉会告诉我们有三种贪心策略。面试时，导师通常会问你：“为什么另外两种策略不行？”

1.  **贪心策略 1：每次挑“最早开始”的会议。**
    * **反例（翻车）：** 如果第一个会议早上 8 点开始，但要开到晚上 8 点。你贪图它开始得早，结果它霸占了整整一天，原本下午能开的 10 个短会全泡汤了。
2.  **贪心策略 2：每次挑“持续时间最短”的会议。**
    * **反例（翻车）：** 假设会议 A (早 8-早 10)，会议 B (早 9-早 11)，会议 C (早 10-中午 12)。虽然 B 的时间最短（2 小时），但如果选了 B，A 和 C 就都开不了了。最优解显然是选 A 和 C（能安排 2 个会议）。
3.  **贪心策略 3（正解）：每次挑“最早结束”的会议。**
    * **为什么对？** 因为在不冲突的前提下，一个会议**结束得越早，留给后续会议的时间窗口就越大**。这就好比你在装箱子，每次都放最不占空间的东西，最后装的总数一定最多。

---

### <a id="toc-007"></a>三、 C++ 核心代码实现（笔试模板）

理解了策略，代码实现其实非常短。核心就两步：**排序 + 线性遍历扫描**。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 定义会议结构体
struct Meeting {
    int start;
    int end;
};

// 1. 贪心策略的灵魂：自定义比较函数，按【结束时间】从小到大排序
bool cmp(const Meeting& a, const Meeting& b) {
    return a.end < b.end;
}

int maxMeetings(vector<Meeting>& meetings) {
    if (meetings.empty()) return 0;

    // 步骤一：按结束时间最早进行排序
    sort(meetings.begin(), meetings.end(), cmp);

    int count = 1; // 至少能安排一个会议（最早结束的那个）
    int current_end = meetings[0].end; // 记录当前占用会议室的最后一个会议的结束时间

    // 步骤二：从第二个会议开始向后扫描
    for (int i = 1; i < meetings.size(); i++) {
        // 【核心判断】：如果当前候选会议的开始时间，晚于或等于上一个会议的结束时间
        // 说明不冲突，果断安排上！
        if (meetings[i].start >= current_end) {
            count++; // 安排成功，数量 +1
            current_end = meetings[i].end; // 更新会议室被占用的结束时间
        }
    }

    return count;
}
```

### <a id="toc-008"></a>四、 复杂度与考点剖析

* **时间复杂度：** $O(N \log N)$。因为核心消耗在 `sort` 排序上。后面的 `for` 循环遍历只需要 $O(N)$。整体由排序决定。
* **空间复杂度：** $O(1)$ 或 $O(\log N)$（取决于 C++ 底层 `sort` 的实现，通常是原地排序，只需要常数级额外空间记录变量）。

**💡 考场避坑指南（复试加分项）：**
当你在试卷上写完这段代码，或者在面试时口述完这个逻辑后，如果能主动补充一句：**“这个算法的前提是每次做出的局部最优选择（选最早结束的），永远不会剥夺其他更优组合的可能性，因此局部最优解的累加就是全局最优解。”** 这会让导师立刻觉得你的理论基础非常扎实。

这个会议室调度的逻辑你觉得好消化吗？如果你已经完全掌握，我们可以看看贪心算法的另一个硬核变种——**“跳跃游戏 (Jump Game)”**，或者你想转向图论中基于贪心的 **Dijkstra 最短路径算法**？

## <a id="toc-009"></a>贪心算法证明题

如果背包问题中，所有的物品价值相同，但是重量不同，那么动态规划问题可以使用贪心算法解决。

### <a id="toc-010"></a>一、 算法描述与时间复杂度 

**1. 算法简述 (Algorithm Description):**
由于所有物品价值相同，问题转化为求在容量 $W$ 限制下，能装入的最多的物品数量。
贪心策略如下：
* **Step 1:** 将这 $n$ 个物品按照**重量从轻到重**进行升序排序。
* **Step 2:** 初始化当前背包重量为 $0$，已选物品数量为 $0$。
* **Step 3:** 按照排序后的顺序，依次尝试将物品放入背包。如果当前物品的重量加上背包已有重量没有超过 $W$，则将其放入，并更新背包重量和物品数量；否则，结束放入过程。

**2. 时间复杂度 (Time Complexity):**
* 排序步骤需要 $O(n \log n)$ 的时间。
* 遍历选择步骤需要 $O(n)$ 的时间。
* **总时间复杂度为 $O(n \log n)$。**

---

### <a id="toc-011"></a>二、 证明贪心原则 

**核心思路：替换法 (Exchange Argument)**
我们要证明：**“选择最轻的物品”这个决定，一定包含在某个全局最优解中。** 证明方法是假设存在一个最优解没有选它，然后通过“无损替换”证明选了它依然是最优解。

你可以直接将以下逻辑作为你的标准答题模板写在卷面上：

**【证明过程】**
1.  **定义贪心选择：** 假设所有物品中**重量最轻**的物品为 $x$，其重量为 $w_x$。
2.  **假设存在一个最优解没有包含贪心选择：** 假设存在一个最优解集合 $O$，它使得装入的物品数量最多，但集合 $O$ 中**并没有包含**物品 $x$。
3.  **构造替换：** 因为背包容量有限，而集合 $O$ 并非空集（假设 $W$ 足够大至少能装一个），我们从最优解集合 $O$ 中任取一个物品 $y$，其重量为 $w_y$。
4.  由于 $x$ 是所有物品中最轻的，必然有 **$w_x \le w_y$**。
5.  现在，我们将集合 $O$ 中的物品 $y$ 拿出来，替换为最轻的物品 $x$。我们得到一个新的集合 **$O' = O - \{y\} + \{x\}$**。
6.  **验证合法性（不超重）：**
    新集合的总重量为：$Weight(O') = Weight(O) - w_y + w_x$。
    因为 $w_x \le w_y$，所以 $Weight(O') \le Weight(O) \le W$。
    这意味着新集合 $O'$ 依然满足背包的容量限制，是一个合法的解。
7.  **验证最优性（价值不减）：**
    新集合 $O'$ 与原集合 $O$ 包含的物品总数完全一样（都是一出一进），因为所有物品价值相同，所以它们产生的总价值也完全一样。
8.  **得出结论：** 新集合 $O'$ 同样是一个最优解，且它包含了最轻的物品 $x$。这就证明了，**做出第一次贪心选择（选最轻的）总是能导向全局最优解，贪心原则成立。**

---

**💡 考场小贴士：**
以后在算法笔试中，只要题目让你证明“贪心策略为什么是对的”，**90% 都是用这种“替换法”**：
先假设一个不包含贪心选择的“最优解”，然后用你的“贪心选择”去替换掉里面的某个元素，最后证明替换后的新解**既没有违反规则（比如没超重），也没有让结果变差（比如价值没变少）**，证明就完美结束了。
