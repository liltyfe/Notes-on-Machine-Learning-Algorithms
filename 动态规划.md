## <a id="toc-0001"></a>动态规划

### <a id="toc-0002"></a>一、 动态规划的核心本质

动态规划本质上是一种**空间换时间**的算法思想。它并不是一种具体的算法（像快速排序那样），而是一种解决特定类型问题的**策略**。

它的核心在于识别并利用问题的两个基本属性：
1.  **重叠子问题 (Overlapping Subproblems)：** 在穷举的过程中，相同的子问题会被重复计算多次。DP 通过使用内存（通常是数组或哈希表）将这些子问题的结果存储起来（Memoization/Tabulation），避免重复计算。
2.  **最优子结构 (Optimal Substructure)：** 原问题的最优解可以由其子问题的最优解推导出来。这意味着我们可以按顺序自底向上地解决问题。


与你在数据结构中学过的“分治法”不同：分治法的子问题是相互独立的（比如归并排序的两半），而 DP 的子问题是相互重叠的。

---

### <a id="toc-0003"></a>二、 破解 DP 问题的“标准四步曲”

面对任何一道 DP 考题，不要一上来就写代码，而是要在草稿纸上推演以下四个步骤：

1.  **定义状态 (State Definition)：**
    这是最关键也是最难的一步。你需要明确 $dp$ 数组（或矩阵）中的 $dp[i]$ 或 $dp[i][j]$ 到底代表什么物理意义。状态的定义通常需要包含当前所处的阶段以及相关的限制条件。
2.  **推导状态转移方程 (State Transition Equation)：**
    找出当前状态是如何由之前的状态推导过来的。这通常是一个极值问题（求最大/最小）或计数问题。
3.  **确定初始化条件 (Initialization)：**
    确定 $dp$ 数组的最小子问题（边界条件）的值是什么。例如 $dp[0]$ 或 $dp[0][0]$，这是整个多米诺骨牌的第一张。
4.  **确定遍历顺序 (Computation Order)：**
    根据状态转移方程中依赖的前置状态，决定是正向遍历、逆向遍历还是其他更复杂的遍历方式，确保在计算当前状态时，它所依赖的状态已经被计算过了。

---

### <a id="toc-0004"></a>三、 实战推演：经典的 0-1 背包问题

为了让你快速找到手感，我们用复试中最常考的 **0-1 背包问题**来过一遍这四个步骤。

**问题描述：** 给定 $N$ 个物品和一个容量为 $W$ 的背包。第 $i$ 个物品的重量为 $w[i]$，价值为 $v[i]$。每个物品只能选一次（0 或 1），求背包能装入的最大总价值。

1.  **定义状态：**
    设定二维数组 $dp[i][j]$，表示在前 $i$ 个物品中进行选择，且当前背包剩余容量为 $j$ 时，所能获得的最大价值。
2.  **状态转移方程：**
    对于第 $i$ 个物品，我们只有两种选择：
    * **不选第 $i$ 个物品：** 那么最大价值就等同于在前 $i-1$ 个物品中容量为 $j$ 的最大价值。即：$dp[i-1][j]$
    * **选择第 $i$ 个物品：** 前提是剩余容量 $j$ 必须大于等于 $w[i]$。此时的价值等于前 $i-1$ 个物品在容量为 $j - w[i]$ 时的最大价值，加上当前物品的价值 $v[i]$。即：$dp[i-1][j-w[i]] + v[i]$
    
    综合这两种情况，我们取最大值：
    $$dp[i][j] = \max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])$$
3.  **初始化：**
    当物品数量为 $0$ 时，或者背包容量为 $0$ 时，最大价值都是 $0$。所以 $dp[0][...]$ 和 $dp[...][0]$ 全部初始化为 $0$。
4.  **遍历顺序：**
    由于 $dp[i][j]$ 依赖于 $i-1$ 的上一层状态，我们需要外层循环遍历物品 $i$，内层循环遍历容量 $j$（正序逆序皆可，如果在空间优化成一维数组时则必须逆序）。


**核心 C++ 逻辑实现：**
```cpp
for (int i = 1; i <= N; i++) {
    for (int j = 1; j <= W; j++) {
        if (j >= w[i]) {
            dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i]);
        } else {
            dp[i][j] = dp[i-1][j];
        }
    }
}
```

## <a id="toc-0005"></a>动态规划如何保重价值最大（最优解）

### <a id="toc-0006"></a>1. 本质并未放弃“穷举”，而是进行了“状态的折叠”
在算法世界里，想要寻找绝对的最大值，最严谨、永远不会错的方法是什么？是把所有可能的组合全都试一遍。

动态规划**并没有放弃穷举**，它只是聪明地合并了那些殊途同归的废操作。
如果你有 $N$ 个物品，暴力的决策树会有 $2^N$ 个叶子节点（每个物品选或不选）。但在 DP 的二维数组中，我们只关心**“当前到了第几个物品”**以及**“当前背包还剩多少容量”**。
无论前面 $i-1$ 个物品是怎么组合的，只要它们消耗了相同的容量 $k$，并且产生了相同的最大价值 $V$，它们在第 $i$ 步面临的未来是一模一样的。DP 就是把决策树上无数个多余的分支，折叠成了一个个具体的数组格子 $dp[i][j]$。
**既然它的本质依然是覆盖了所有有效组合的穷举，那么它算出来的值，自然就不会漏掉任何一个可能的最大值。**

### <a id="toc-0007"></a>2. 核心底座：“无后效性”（Markov Property）
这是 DP 状态转移能够成立的命门。你可以这样理解无后效性：**“未来与过去无关，只取决于现在。”**

假设你现在推导到了 $dp[5][10] = 50$（前 5 个物品，剩 10 容量，目前最大价值 50）。
为了凑出这 50 的价值，你到底是选了物品 1 和 3，还是选了物品 2 和 4？**这完全不重要了！**
因为 0-1 背包的规则是每个物品只能选一次。前 5 个物品的内部纠葛，绝不会影响你对第 6 个物品的判断。
只要这个“50”确实是前 5 个物品在消耗 10 个容量下的极限战绩，我们就可以放心地把它作为基石，继续去推导第 6 个物品。过去的具体选择路径不会限制未来的决策，这就打破了局部最优无法推导全局最优的魔咒。

### <a id="toc-0008"></a>3. 严密的数学归纳法（多米诺骨牌效应）
我们可以把状态转移方程看作是一个严密的数学归纳证明过程。
状态转移方程：$dp[i][j] = \max(dp[i-1][j], \ dp[i-1][j-w[i]] + v[i])$

* **第一块骨牌（边界）：** $dp[0][j] = 0$。即没有物品可选时，无论容量多大，价值都是 0。这是绝对正确的真理。
* **归纳假设：** 假设第 $i-1$ 行的所有值 $dp[i-1][...]$，都已经完美保存了前 $i-1$ 个物品在各自容量下的绝对最大价值。
* **推导第 $i$ 行：** 对于当前物品 $i$，由于物理法则的限制，你只有“放”和“不放”两条路。
    * 如果你**不放**，最大价值就是 $dp[i-1][j]$。
    * 如果你**放**，最大价值必定是它剩余空间能装下的最大价值 $dp[i-1][j-w[i]]$，加上当前物品的价值 $v[i]$。
    方程对这两条唯一可能的合法路径取了 `max()`。

既然“地基”（第 0 行）是绝对正确的，且“砌砖”的规则（每一次都严格对比所有合法前提下的最大值）也是无懈可击的，那么一层层推导出来的顶层结果 $dp[N][W]$，在数学逻辑上必定是全局最大值。


### <a id="toc-0009"></a>4. 图论视角的降维打击

如果你把这二维数组里的每一个格子 $dp[i][j]$ 看作有向图中的一个顶点，把“选”与“不选”看作带有权重的有向边，那么整个 0-1 背包问题，实际上就是**在一张有向无环图（DAG）中，寻找从起点 $dp[0][W]$ 到任意终点的最长路径**。

在这个图里，外层循环 $i$ 保证了图是单向的（不会产生环路）。状态转移其实就是对图进行了一次拓扑排序后的最长路径计算（类似于在网格中找最长路径）。既然每一步都在汇聚的边中挑选了最长的那条走过来，最后抵达终点的，必然是整个图的全局最长路径。


## <a id="toc-0010"></a>常见的 DP 模型

在接下来的几天里，建议你按照以下模型进行分类刷题和总结：
* **线性 DP：** 最长递增子序列 (LIS)、最长公共子序列 (LCS)、最大子数组和。
* **背包 DP：** 0-1 背包、完全背包、多重背包（以及它们的空间优化方法，如滚动数组）。
* **区间 DP：** 矩阵连乘、石子合并问题。
* **树形 DP / 状态压缩 DP：** 这类相对高阶，如果报考的是顶尖院校或核心实验室，可能会在附加题中出现。

## <a id="toc-0011"></a>例题一

### <a id="toc-0012"></a>一、 题目重述

**三、 （15分）动态规划（矩阵连乘问题）**

给定一个矩阵子序列 $A_i A_{i+1} ... A_j$，其维度分别为 $p_{i-1} p_i ... p_j$，其中 $A_x$ 的维度为 $p_{x-1} \times p_x$。我们试图确定最佳的相乘顺序，以获得最小的乘法操作次数。

**(1)** 令 $m[i, j]$ 表示计算上述序列所需的最小乘法次数。请填写以下关于 $m[i, j]$ 的递归公式。（5分）
$$m[i, j] = \begin{cases} \text{\_\_\_\_\_\_\_\_\_\_\_\_\_\_} & i == j \\ \text{\_\_\_\_\_\_\_\_\_\_\_\_\_\_} & i < j \end{cases}$$

**(2)** 我们有以下给定的矩阵序列：

| matrix | A1 | A2 | A3 | A4 | A5 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **dimension** | 30 x 35 | 35 x 10 | 10 x 5 | 5 x 15 | 15 x 20 |

填写表格并根据你的计算写出**最佳括号打法 (best parenthesization)**，例如 $(A_1 A_2)((A_3 A_4)A_5)$。

---

### <a id="toc-0013"></a>二、 最终解答


**(1) 递归公式 (状态转移方程):**
$$m[i, j] = \begin{cases} 0 & i == j \\ \min_{i \le k < j} \{ m[i, k] + m[k+1, j] + p_{i-1} p_k p_j \} & i < j \end{cases}$$

**(2) 填表结果与最佳括号打法:**

**完整的 m 矩阵 (最小乘法次数) 表格如下：**

| $i \backslash j$ | 1 | 2 | 3 | 4 | 5 |
| :---: | :---: | :---: | :---: | :---: | :---: |
| **1** | 0 | 10500 | 7000 | 9250 | 11500 |
| **2** | - | 0 | 1750 | 4375 | 6750 |
| **3** | - | - | 0 | 750 | 2500 |
| **4** | - | - | - | 0 | 1500 |
| **5** | - | - | - | - | 0 |

**最佳括号打法 (Best Parenthesization):**
$$((A_1(A_2 A_3))(A_4 A_5))$$

---

### <a id="toc-0014"></a>三、 详细解析过程 (解题思路与草稿)

面对区间 DP，我们的核心思路是**枚举区间长度**，从小区间慢慢推导到大区间。

**第一步：提取维度数组 $p$**
根据题意，$A_k$ 的维度是 $p_{k-1} \times p_k$。我们先把题目表格中的维度转化为一个一维数组 $p$，这是计算的基础：
* $A_1$ 是 $30 \times 35 \implies p_0 = 30, p_1 = 35$
* $A_2$ 是 $35 \times 10 \implies p_2 = 10$
* $A_3$ 是 $10 \times 5 \implies p_3 = 5$
* $A_4$ 是 $5 \times 15 \implies p_4 = 15$
* $A_5$ 是 $15 \times 20 \implies p_5 = 20$
**数组 $p = [30, 35, 10, 5, 15, 20]$**

**第二步：按区间长度 $L$ 逐层计算**
为了推导出最优解，除了记录最小代价的 $m$ 矩阵，我们在草稿纸上还需要记录一个**分割点矩阵 $s[i, j]$**（记录在哪个 $k$ 值处取得了最小值），这用于最后反推括号打法。

* **区间长度 $L=1$ (对角线):** 不需要相乘，$m[i,i] = 0$。（已在考卷上填好）

* **区间长度 $L=2$ (相邻两个矩阵相乘):** 只有一种切分方法 $k=i$。
    * $m[1, 2] = m[1, 1] + m[2, 2] + p_0 p_1 p_2 = 0 + 0 + 30 \times 35 \times 10 = \mathbf{10500}$  (在 $k=1$ 处断开)
    * $m[2, 3] = m[2, 2] + m[3, 3] + p_1 p_2 p_3 = 0 + 0 + 35 \times 10 \times 5 = \mathbf{1750}$  (在 $k=2$ 处断开)
    * $m[3, 4] = m[3, 3] + m[4, 4] + p_2 p_3 p_4 = 0 + 0 + 10 \times 5 \times 15 = \mathbf{750}$   (在 $k=3$ 处断开)
    * $m[4, 5] = m[4, 4] + m[5, 5] + p_3 p_4 p_5 = 0 + 0 + 5 \times 15 \times 20 = \mathbf{1500}$  (在 $k=4$ 处断开)

* **区间长度 $L=3$ (三个矩阵相乘):** 需要找一个切分点 $k$ 使得代价最小。
    * **计算 $m[1, 3]$**：切分点 $k$ 可以是 $1$ 或 $2$。
        * $k=1 \implies (A_1)(A_2 A_3) \implies m[1, 1] + m[2, 3] + p_0 p_1 p_3 = 0 + 1750 + 30 \times 35 \times 5 = 7000$
        * $k=2 \implies (A_1 A_2)(A_3) \implies m[1, 2] + m[3, 3] + p_0 p_2 p_3 = 10500 + 0 + 30 \times 10 \times 5 = 12000$
        * 取最小值，所以 $m[1, 3] = \mathbf{7000}$，记录最佳切分点 $s[1, 3] = 1$。
    * **计算 $m[2, 4]$**:
        * $k=2 \implies m[2, 2] + m[3, 4] + p_1 p_2 p_4 = 0 + 750 + 35 \times 10 \times 15 = 6000$
        * $k=3 \implies m[2, 3] + m[4, 4] + p_1 p_3 p_4 = 1750 + 0 + 35 \times 5 \times 15 = 4375$
        * 最小值 $m[2, 4] = \mathbf{4375}$，最佳切分点 $s[2, 4] = 3$。
    * **计算 $m[3, 5]$**:
        * $k=3 \implies m[3, 3] + m[4, 5] + p_2 p_3 p_5 = 0 + 1500 + 10 \times 5 \times 20 = 2500$
        * $k=4 \implies m[3, 4] + m[5, 5] + p_2 p_4 p_5 = 750 + 0 + 10 \times 15 \times 20 = 3750$
        * 最小值 $m[3, 5] = \mathbf{2500}$，最佳切分点 $s[3, 5] = 3$。

* **区间长度 $L=4$:**
    * **计算 $m[1, 4]$**:
        * $k=1 \implies 0 + 4375 + (30 \times 35 \times 15) = 20125$
        * $k=2 \implies 10500 + 750 + (30 \times 10 \times 15) = 15750$
        * $k=3 \implies 7000 + 0 + (30 \times 5 \times 15) = 9250$
        * 最小值 $m[1, 4] = \mathbf{9250}$，切分点 $s[1, 4] = 3$。
    * **计算 $m[2, 5]$**:
        * $k=2 \implies 0 + 2500 + (35 \times 10 \times 20) = 9500$
        * $k=3 \implies 1750 + 1500 + (35 \times 5 \times 20) = 6750$
        * $k=4 \implies 4375 + 0 + (35 \times 15 \times 20) = 14875$
        * 最小值 $m[2, 5] = \mathbf{6750}$，切分点 $s[2, 5] = 3$。

* **区间长度 $L=5$ (终极目标):**
    * **计算 $m[1, 5]$**:
        * $k=1 \implies m[1,1] + m[2,5] + p_0 p_1 p_5 = 0 + 6750 + (30 \times 35 \times 20) = 27750$
        * $k=2 \implies m[1,2] + m[3,5] + p_0 p_2 p_5 = 10500 + 2500 + (30 \times 10 \times 20) = 19000$
        * $k=3 \implies m[1,3] + m[4,5] + p_0 p_3 p_5 = 7000 + 1500 + (30 \times 5 \times 20) = \mathbf{11500}$
        * $k=4 \implies m[1,4] + m[5,5] + p_0 p_4 p_5 = 9250 + 0 + (30 \times 15 \times 20) = 18250$
        * 得出最终答案 $m[1, 5] = 11500$，且最高层切分点 **$s[1, 5] = 3$**。

**第三步：回溯构造最佳括号打法**
我们利用刚才记录的切分点 $s$ 矩阵，从大到小“撕开”这个序列：
1.  我们要解 $(A_1 ... A_5)$。根据 $s[1, 5] = 3$，序列在 $A_3$ 和 $A_4$ 之间断开。
    $\implies (A_1 .. A_3) (A_4 A_5)$
2.  看左边子问题 $(A_1 .. A_3)$。根据前面算出的 $s[1, 3] = 1$，序列在 $A_1$ 和 $A_2$ 之间断开。
    $\implies (A_1) (A_2 A_3)$
3.  看右边子问题 $(A_4 A_5)$。它只有两个元素，自然是 $(A_4 A_5)$。
4.  将所有步骤组合起来：左边是 $(A_1(A_2 A_3))$，右边是 $(A_4 A_5)$。
    最终结合：**$((A_1(A_2 A_3))(A_4 A_5))$**

### <a id="toc-0015"></a>关于题解的一些注解

#### <a id="toc-0016"></a>一、 这道题到底在让我们干什么？

题目的核心诉求是：**寻找一种加括号的方式，让几个矩阵连乘的“标量乘法总次数”最少。**

**前置知识（复习一下线性代数）：**
当两个矩阵相乘时，比如矩阵 $A$ 的维度是 $M \times N$，矩阵 $B$ 的维度是 $N \times P$，它们相乘得到的新矩阵维度是 $M \times P$。
而计算机在底层计算这个新矩阵时，需要进行的乘法运算总次数是：$M \times N \times P$。

**为什么加括号的顺序会影响计算量？**
矩阵乘法满足结合律（先算谁都行，最终结果矩阵的数值是一样的），但是**计算代价**天差地别。

给你举一个极端的例子。假设有三个矩阵：
* $A$: $10 \times 100$
* $B$: $100 \times 5$
* $C$: $5 \times 50$

现在我们要计算 $A \times B \times C$，有两种加括号的策略：

**策略一：先算 $(A \times B)$，再乘 $C$**
1.  算 $A \times B$：乘法次数 = $10 \times 100 \times 5 = 5000$ 次。得到一个新矩阵，维度是 $10 \times 5$。
2.  拿新矩阵乘 $C$：乘法次数 = $10 \times 5 \times 50 = 2500$ 次。
3.  **总代价 = $5000 + 2500 = 7500$ 次。**

**策略二：先算 $(B \times C)$，拿 $A$ 去乘它**
1.  算 $B \times C$：乘法次数 = $100 \times 5 \times 50 = 25000$ 次。得到一个新矩阵，维度是 $100 \times 50$。
2.  拿 $A$ 乘新矩阵：乘法次数 = $10 \times 100 \times 50 = 50000$ 次。
3.  **总代价 = $25000 + 50000 = 75000$ 次。**

你看，仅仅是换了一下计算顺序，计算量相差了整整 10 倍！
**这道题的意思就是：现在给你 5 个矩阵，你写个程序（或者手动推导），告诉我应该怎么加括号，才能让计算机干的活儿最少。**

而这里的 $m[i,j]$，表示从$A_i$乘到$A_j$所需要的最小的乘法数量。

#### <a id="toc-0017"></a>二、 为什么要提取成一个一维数组？

这纯粹是一个**数据结构上的压缩技巧**。

你看题目给的表格：
$A_1$ 是 30 x **35**
$A_2$ 是 **35** x 10
$A_3$ 是 10 x 5

矩阵能够相乘的铁律是：**前一个矩阵的列数，必须等于后一个矩阵的行数**。所以 $A_1$ 结尾的 35，必定等于 $A_2$ 开头的 35。

如果在写代码时，我们用二维数组去存 `[[30, 35], [35, 10], [10, 5]]`，不仅浪费内存，而且存在大量重复数据（两个 35，两个 10）。

为了让公式和代码更干净，我们把这些重复相邻的数字合并，像串糖葫芦一样串成一个一维数组：
数组 $p = [30, 35, 10, 5, ...]$
这样规定之后，任何一个矩阵 $A_k$ 的维度，都可以直接用 $p[k-1] \times p[k]$ 来表示。比如 $A_2$ 的维度就是 $p[1] \times p[2]$，即 $35 \times 10$。

这并不是什么高深的算法，只是为了方便后续写那个乘法公式 $p_{i-1} \times p_k \times p_j$。

#### <a id="toc-0018"></a>三、 “按区间长度 $L$ 逐层计算”是什么意思？

这是这道题最核心的**动态规划推导逻辑**。

你想求 5 个矩阵相乘的最优解，能不能直接一眼看出来？不能。那怎么办？我们从小到大，像搭积木一样拼出来。这里的“区间长度 $L$”指的就是**连在一起相乘的矩阵个数**。

1.  **当 $L=1$ 时（自己乘自己）：**
    比如就一个矩阵 $A_1$。不需要任何乘法操作，代价是 0。这就是公式里写的 `i == j` 时代价为 0。（这就是打地基）
2.  **当 $L=2$ 时（两个相邻矩阵相乘）：**
    比如 $A_1 \times A_2$。只有一种乘法，没得选，代价就是 $30 \times 35 \times 10$。我们在这一层把所有两个相邻矩阵的结果都算出来并存好。
3.  **当 $L=3$ 时（三个相邻矩阵相乘，开始面临选择了！）：**
    比如要算 $A_1 \times A_2 \times A_3$。这时候要在中间“切一刀”（找一个分割点 $k$）：
    * 切法一：$(A_1) \times (A_2 A_3)$。这需要知道 $(A_2 A_3)$ 的最优解。去哪里找？在 $L=2$ 那一层我们已经算好存起来了！
    * 切法二：$(A_1 A_2) \times (A_3)$。这需要知道 $(A_1 A_2)$ 的最优解，同样在 $L=2$ 那一层已经有了。
    我们把这两种切法算出来的总代价比大小，把更小的那个记录下来。

以此类推，当你想算最终的 $L=5$（全部 5 个矩阵，即 $m[1, 5]$）时，你需要尝试把它切分成两块，比如分成左边 2 个和右边 3 个，或者左边 1 个和右边 4 个。这就**必须依赖之前算好的短区间的结果**。

所以，“按区间长度 $L$ 逐层计算”的意思就是：**你不能上来就填表格的右上角（整体结果），你必须斜着填表，先算所有 2 个矩阵的组合，再算所有 3 个矩阵的组合，一步步扩大范围，直到算出整条链条的最优解。**

## <a id="toc-0019"></a>例题二：最长回问子序列

这题和矩阵连乘的底层逻辑一模一样：**从小区间（短字符串）推导到大区间（长字符串）**。不能从左往右线性推导，必须按“区间长度”斜着填表。

---

### <a id="toc-0020"></a>1. 完善状态转移方程 (Recursive Formula)

这道题考察的是区间 DP 的状态推导。$L[i, j]$ 代表从字符 $i$ 到字符 $j$ 的最长回文子序列长度。

**答案填写：**
$$L[i, j] = \begin{cases} 1 & i = j \\ \mathbf{2 \ (\text{若 } A[i] = A[j], \text{ 否则为 } 1)} & i + 1 = j \\ \mathbf{L[i+1, j-1] + 2} & A[i] = A[j] \\ \mathbf{\max(L[i, j-1], L[i+1, j])} & A[i] \neq A[j] \end{cases}$$

**逻辑拆解（为什么这么填）：**
* **第 1 行 ($i=j$)：** 只有一个字符，回文长度必然是 1。这就是我们的“地基”。
* **第 2 行 ($i+1=j$)：** 只有两个相邻字符（区间长度为 2）。出题人把这个单独拎出来，是因为如果在长度为 2 时套用下面的公式，会出现 $L[i+1, j-1]$，即 $L[i+1, i]$，这会导致左指针大于右指针，越界了。所以要单独处理：如果俩字符相等，长度就是 2；不相等就是 1。*(注：严谨起见，建议在卷面上把相等和不相等的情况都写上)*。
* **第 3 行 ($A[i]=A[j]$)：** 左右两端的字符相等！它们俩直接为回文贡献了 2 的长度。剩下的就看被它们夹在中间的子串 $A[i+1 \dots j-1]$ 能贡献多长的回文。
* **第 4 行 ($A[i] \neq A[j]$)：** 左右两端不相等，说明它们不可能同时出现在最长回文里。我们要么舍弃左端点（看 $L[i+1, j]$），要么舍弃右端点（看 $L[i, j-1]$），两者取最大值。


---

### <a id="toc-0021"></a>2. 编写算法代码 (Program implementation)

既然这是标准的区间 DP，我们就直接默写区间 DP 的“三层循环模板”：第一层枚举区间长度，第二层枚举左端点，第三层推导右端点。代码风格非常清晰鲁棒。

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

int longestPalindromeSubseq(string s) {
    int n = s.length();
    // 定义状态矩阵 L，初始化为 0
    vector<vector<int>> L(n, vector<int>(n, 0));

    // Base Case: 长度为 1 的区间 (对应 i == j)
    for (int i = 0; i < n; i++) {
        L[i][i] = 1;
    }

    // 核心逻辑：按区间长度从短到长进行计算 (完美对应矩阵连乘的填表顺序)
    for (int len = 2; len <= n; len++) {
        for (int i = 0; i < n - len + 1; i++) {
            int j = i + len - 1; // 计算右端点 j
            
            if (s[i] == s[j]) {
                if (len == 2) {
                    L[i][j] = 2; // 对应 i + 1 == j 的情况
                } else {
                    L[i][j] = L[i+1][j-1] + 2; 
                }
            } else {
                L[i][j] = max(L[i+1][j], L[i][j-1]);
            }
        }
    }
    
    // 最终答案是从第一个字符到最后一个字符的最长回文长度
    return L[0][n-1]; 
}
```

### <a id="toc-0022"></a>3. 分析计算复杂度 (Computational Complexity)

这道题的复杂度分析非常直观，可以直接写在卷面上：

* **时间复杂度 (Time Complexity): $O(n^2)$**
    **分析：** 算法主要包含两层嵌套循环（外层循环控制子串长度 `len` 从 2 到 $n$，内层循环控制起始位置 `i`）。这相当于填充一个 $n \times n$ 矩阵的右上半部分。总共需要计算约 $\frac{n(n-1)}{2}$ 个状态。每次状态转移只需要 $O(1)$ 的比较和加法操作。因此整体时间复杂度为 $O(n^2)$。
* **空间复杂度 (Space Complexity): $O(n^2)$**
    **分析：** 算法开辟了一个 $n \times n$ 的二维数组（或者矩阵）`L` 来存储所有子问题的最优解状态，因此空间复杂度为 $O(n^2)$。*(注：如果想在老师面前秀一下，可以补充说明：由于当前状态只依赖于它的左边、下边和左下角的状态，可以通过一维数组/滚动数组将其空间复杂度优化至 $O(n)$。)*

这道题其实就是把矩阵连乘的“寻找切分点 $k$” 换成了“判断左右两端是否相等”。你看，当你彻底掌握了**“区间 DP 就是按长度从小到大枚举”**这个底层逻辑后，这类应用题就是换汤不换药的送分题。
